# tooling/shared/ — Canonical Dev and Version Scripts

## Purpose

Source-of-truth development and versioning scripts. Every Obsidian plugin in the ecosystem maintains identical copies via `sync:plugins` propagation. These scripts are never edited directly in downstream plugins.

| File | Purpose |
|------|---------|
| `dev.mjs` | Vault selector, esbuild watch mode, hot-reload coordination with Obsidian |
| `version.mjs` | Semver bump (patch/minor/major), manifest.json update, changelog generation |

## Script Behavior

**dev.mjs:**
- Interactive vault selection on first run
- Watches src/ for changes
- Triggers esbuild rebuild
- Reloads plugin via Obsidian API
- Stores vault path in `.env.local`

**version.mjs:**
- Reads current version from manifest.json
- Prompts for semver bump (patch/minor/major)
- Updates manifest.json
- Generates/appends to CHANGELOG.md
- Updates package.json version
- Triggers git commit and tag

## Integration

Used by npm scripts in package.json:
- `pnpm dev` → runs `dev.mjs`
- `pnpm release:patch|minor|major` → runs `version.mjs`

CI workflows invoke these scripts as part of release pipeline.
